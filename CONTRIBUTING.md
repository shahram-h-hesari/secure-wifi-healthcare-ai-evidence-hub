# Contributing to the AI/ML WiFi Sensing Hub

Thank you for your interest in contributing to the **AI/ML WiFi Sensing Hub**.

This repository is an open research mapping hub for organizing papers, datasets, code releases, reproducibility notes, security gaps, and application categories in AI/ML WiFi sensing. The first focus area is healthcare-relevant sensing, including contactless monitoring, activity recognition, fall detection, respiration sensing, and care-aware environments.

---

## Ways to Contribute

You can contribute by suggesting:

- Research papers
- Public datasets
- Code repositories
- Reproducibility notes
- Security, privacy, or robustness gaps
- Research groups, labs, or collaboration opportunities
- New AI/ML WiFi sensing application categories

Please use the GitHub issue forms when possible so each suggestion can be reviewed consistently.

---

## Suggesting a Paper

Use the **Suggest a Paper** issue form.

Helpful information includes:

- Paper title
- Authors
- Year and venue
- DOI, arXiv, publisher, or project link
- Application category
- Dataset used
- Code availability
- Security, robustness, or privacy relevance
- Reason the paper should be included

---

## Suggesting a Dataset

Use the **Suggest a Dataset** issue form.

Helpful information includes:

- Dataset name
- Official dataset link
- Related papers
- Data type or modality
- Access status
- License or usage notes
- Application category
- Relevance to AI/ML WiFi sensing

## Suggesting a Code Repository

Use the **Suggest a Code Repository** issue form.

## Suggesting Collaboration Leads

Use the **Collaboration Interest** issue form.

Relevant candidates may include:

- Authors of related papers
- Research labs
- Dataset owners
- Code maintainers
- Healthcare AI groups
- Wireless sensing groups
- Security and privacy researchers
- Smart-environment or aging-technology communities

---

## Scope

This hub currently focuses on:

- AI/ML WiFi sensing
- WiFi CSI and wireless sensing
- Healthcare-relevant sensing
- Human activity recognition
- Contactless monitoring
- Security, privacy, and robustness
- Reproducible research
- Dataset and code availability

Future categories may include smart environments, robotics, IoT sensing, industrial sensing, and broader WiFi-based AI sensing applications.

---

## Out of Scope

The following are currently out of scope:

- Unsupported clinical claims
- Medical-device validation claims
- Private patient data
- Unlicensed dataset redistribution
- Unrelated general AI papers with no WiFi sensing connection
- General wireless communication papers with no sensing, AI/ML, security, or reproducibility connection
- Marketing-only content without research evidence

---

## Review Process

Suggested items may be reviewed for:

- Relevance to AI/ML WiFi sensing
- Application category
- Dataset availability
- Code availability
- Reproducibility value
- Security, robustness, or privacy relevance
- Evidence strength
- Collaboration value

After review, items may be added to the evidence map, moved to the GitHub Project board, or archived as out of scope.

---

## Research Disclaimer

This repository is a research evidence hub. It does **not** claim clinical validation, medical-device readiness, real patient deployment, regulatory approval, or diagnostic capability.

The focus is scientific evidence mapping, reproducibility, security analysis, and trustworthy AI/ML sensing methods for WiFi-based sensing applications.
