# AI/ML WiFi Sensing Hub Roadmap

This roadmap outlines the planned development of the **AI/ML WiFi Sensing Hub** as an open research mapping resource for WiFi sensing papers, datasets, code releases, reproducibility notes, security gaps, and application categories.

---

## Current Focus

The first focus area is **healthcare-relevant WiFi sensing**, including:

- Contactless activity recognition
- Fall detection
- Respiration and vital-sign-related sensing
- Aging-in-place and assisted-living research
- Care-aware environments
- Safety-aware sensing evaluation

---

## Development Phases

| Phase | Goal | Status |
|---|---|---|
| Phase 1 | Establish repository identity, README, project board, labels, and contribution pathway | In Progress |
| Phase 2 | Build initial healthcare-relevant paper intake list | Planned |
| Phase 3 | Create dataset and code availability index | Planned |
| Phase 4 | Map security, robustness, and privacy evidence | Planned |
| Phase 5 | Add reproducibility and benchmark review notes | Planned |
| Phase 6 | Identify research groups, labs, and collaboration candidates | Planned |
| Phase 7 | Expand into additional AI/ML WiFi sensing categories | Future |

---

## Planned Evidence Resources

| Resource | Purpose |
|---|---|
| `papers_index.csv` | Structured list of reviewed papers |
| `datasets_index.csv` | Dataset access, modality, license, and experiment-suitability tracking |
| `code_releases_index.csv` | Code availability, setup clarity, and reproducibility notes |
| `security_gap_map.csv` | Security, privacy, adversarial ML, and robustness evidence |
| `collaboration_candidates.csv` | Authors, labs, dataset owners, and communities connected to the hub |

---

## Near-Term Priorities

1. Add the first 10 healthcare-relevant WiFi sensing papers.
2. Identify public datasets and code repositories linked to those papers.
3. Mark dataset/code availability and reproducibility status.
4. Map security, robustness, and privacy relevance.
5. Identify research groups or labs connected to high-value papers and datasets.

---

## Future Expansion Areas

Potential future categories include:

- Smart environments
- Human activity recognition
- Security and privacy
- Robotics and device-free interaction
- Industrial and IoT sensing
- Reproducible WiFi sensing benchmarks

---

## Research Positioning

The hub is intended to support structured scientific evidence mapping and collaboration across AI/ML, WiFi sensing, wireless security, reproducible research, and healthcare-relevant sensing applications.

It does **not** claim clinical validation, medical-device readiness, real patient deployment, regulatory approval, or diagnostic capability.
